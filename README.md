## Who I Am

I build **AI-native product systems** on the .NET stack — knowledge platforms, GraphRAG pipelines, multi-agent workflows, Orleans infrastructure, and the developer tooling around them. 20+ years in the industry: started as a sysadmin, spent a decade writing C# for other people, then decided it was time to write it for myself.

Today I run **[Managed Code](https://www.managed-code.com)** — a boutique AI solutions agency in France that works with clients where "it broke in production" is not an option. We do clean architecture, AI integration, and distributed systems for companies that take their software seriously.

Along the way I wrote a **[book](https://www.amazon.com/dp/B0CTH5DH95)**, shipped a few dozen open-source libraries, and somehow survived French bureaucracy. Twice.

Made in Ukraine 🇺🇦. Building from Pau, France 🇫🇷.

## What I'm Building

🧠 **[AIBase](https://aibase.fr/)** — enterprise knowledge platform where AI agents don't just chat — they remember context, make decisions, and execute real business workflows. The kind of system your company will wish it had built two years ago.

⚙️ **[MCAF](https://mcaf.managed-code.com/)** — a framework for building software *with* AI coding agents — not a toy demo, but a structured approach with repository memory, explicit rules, tests, and repeatable delivery.

## How I Think About Software

> **Production-ready code from the first commit.**

I've seen enough "we'll refactor it later" turn into "we'll rewrite it never" to know how this works. Software should be engineered as if it will run in production tomorrow — because it usually does.

Clear architecture. Explicit boundaries. Meaningful names. Strong typing. Thorough tests. The discipline that keeps a system coherent when complexity pushes in from every direction.

Engineering shortcuts accumulate interest faster than any business debt. Fragile systems never recover their credibility. If the code cannot handle real load, real change, and real responsibility — it is not finished.

**Quality is not a nice-to-have. Quality *is* the product.**
